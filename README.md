# 🌲 フォレストGIS

**山間部での森林調査を効率化するモバイルアプリ**

[![Flutter](https://img.shields.io/badge/Flutter-3.0+-blue.svg)](https://flutter.dev)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

---

## 🎯 プロダクト目的

- 圏外の森林現場でも止まらない「オフラインファーストGIS」を提供する
- 調査記録（地物・写真・属性）を現場で完結し、帰所後に確実に同期する
- 現場班が短時間で使えるUXで、調査コストと入力ミスを下げる

---

## ✅ 現在の実装状況（2026-05時点）

### 実装済み
- オフライン前提のローカル保存（Drift/SQLite）
- 地図記録（ポイント・ライン・ポリゴン）
- GPS現在地表示、追尾モード、トラック記録
- 写真添付、CSV/GPX/地図画像エクスポート
- Supabase同期（Push/Pull）
- ローカルDBバックアップ/復元
- 同期ステータス詳細（未同期件数・競合ログ・再送キュー）
- 追跡モード切替（高精度/標準/省電力）
- 調査テンプレート入力（毎木/境界/路網）と必須入力チェック
- オフライン地図キャッシュ計画（ズーム範囲・概算容量）

### 未実装・改善余地
- オフライン地図キャッシュ実体（タイル実データ保存/参照）
- 同期競合の解決フロー（ローカル優先/リモート優先選択）
- テンプレート定義の外部化（管理画面/設定ファイル）

## 🧭 市場分析（引継ぎ用サマリ）

### 主要競合
- Avenza Maps（オフライン地図 + GPSの現場運用）
- ArcGIS Field Maps / Survey123（大規模組織導入・統合力）
- QField / QFieldCloud（QGIS資産とOSS運用）
- Fulcrum / TerraFlex（現場入力ワークフローの成熟）

### 共通顧客ニーズ
1. 圏外でも地図と記録が確実に使えること
2. 入力品質を担保するフォーム（必須・条件分岐・辞書）
3. 後工程連携しやすい出力/同期の信頼性
4. バッテリー負荷が低く、学習コストが低いUX

### 勝ち筋（本プロダクト）
- 日本の森林現場に特化したテンプレートUX
- 完全オフライン運用時の信頼性（再送/可視化/競合対処）
- 重厚な基幹連携より「導入の軽さと立上げ速度」で勝つ

## 🔧 Phase 1 実装ガイド（他AI引継ぎ用）

### タスク1: オフライン地図キャッシュ
- `lib/features/map/` にタイル管理モジュールを追加
- 作業エリア画面から「事前ダウンロード」導線を追加
- DBにキャッシュメタ（エリアID、ズーム、容量、更新日時）を追加

### タスク2: 入力テンプレート
- `attribute_editor_dialog.dart` で調査テンプレート選択を追加
- Domain層にテンプレートモデルを追加（必須・選択肢・条件分岐）
- CSV出力をテンプレ項目対応に拡張

### タスク3: 同期可視化（先行着手済み）
- `sync_repository.dart` に `SyncOverview` を追加（未同期件数/競合件数）
- `backup_settings_page.dart` に同期ステータスカードを追加
- 次段で「同期詳細画面」「手動再送」を追加

### タスク4: バッテリー最適化
- 位置更新頻度プリセット（高精度/標準/省電力）
- トラック記録時のみ高頻度化

## 🛠️ 開発者向け

### 技術スタック
- **フレームワーク**: Flutter (Dart)
- **状態管理**: Riverpod
- **地図**: flutter_map + OpenStreetMap
- **ローカルDB**: Drift (SQLite)
- **バックエンド**: Supabase (PostgreSQL + PostGIS)

### セットアップ
```bash
flutter pub get
dart run build_runner build
flutter run
```
